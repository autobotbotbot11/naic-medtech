What This Import Does

1. Reads physicians from `Requesting Physician` rows.
2. Reads rooms from `Room` rows.
3. Reads medtech/pathologist names and license numbers from signatory rows.
4. Creates missing records and reactivates matching inactive records.
5. Keeps existing records instead of deleting or blindly renaming them.

Safety Rules

1. Existing master data is not deleted.
2. Existing signatory licenses are only filled when blank.
3. Conflicting signatory license numbers are kept as warnings for manual review.
4. After import, use the management pages to clean up any clinic-confirmed corrections.
